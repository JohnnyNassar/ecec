01

How it works

The Why ECEC page uses a custom shortcode [ecec_why] that renders five pillars as a sticky-left + scrolling-right scrollytelling layout.

The pillars are hardcoded in the theme's functions.php (not in the WP page editor). To change pillar text or images, you edit the PHP file.

!
Why is it in code, not the editor?

The scrollytelling pattern needs precise HTML structure (sticky containers, IntersectionObserver targets) that Elementor can't reproduce. If you frequently change content, ask the developer to wire it to a custom CPT instead.

Swapping pillar images

Each pillar has an image field in functions.php. Leave it empty to show the gray placeholder, or paste an image URL to display a real photo.

UploadWP Admin → Media → Add New. Drag your image in.
Copy the URLClick the uploaded image. The File URL field on the right shows the full URL — click Copy URL to clipboard.
Open functions.phpPath: wp-content/themes/emaurri-child/functions.php. Search for function ecec_why_shortcode.
Find the $pillars arrayFive rows, one per pillar. Each has 'num', 'title', and 'image' => ''.
Paste the URLReplace the empty string. Example:
'image' => 'http://localhost/ecec/wp-content/uploads/2026/04/why-01.jpg'
Save & deploySave the file. On VPS, sync the theme via the deploy script and hard-refresh the live page.
✓
Recommended specs:

1300 × 848 landscape JPG, ~80% quality. The bleed-right scrolling layout is designed around this aspect ratio. Filenames lowercase with dashes (e.g. why-01.jpg).

i
Reverting to the placeholder:

Set 'image' => '' (empty string) for that pillar — the gray "Image placeholder" box returns automatically.

04

Changing the CTA buttons

Each pillar has an "Explore" CTA that links to /projects/ by default. To change a single pillar's CTA destination, edit the cta_url value for that pillar in functions.php.

To change the CTA text globally (e.g. "Explore" → "Read more"), find the line that outputs the link — it has a hardcoded Explore word. Replace and save.

05

FAQ

Can I add a 6th pillar?

Yes. In the $pillars array, copy one of the existing entries and update the title/body/image. The CSS automatically lays out any number of pillars.

The page is empty / blank.

Check that _elementor_edit_mode postmeta on page 422 is set to builder. Without it, Elementor doesn't render the wrapper section.

Sticky columns don't stick on my browser.

Make sure position: sticky isn't broken by a parent overflow: hidden. The CSS already has overrides for the page wrapper — if you've added new wrappers, they may need overflow: visible.
