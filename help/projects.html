01

Orientation

Each project is one page on the public site. The team at ECEC has built a custom block system that lets you compose any project page out of seven re-usable building blocks — no Elementor required.

i
Where to start.

Open WP Admin → Portfolio in the left sidebar. You'll see the full list of all projects. Click any title to enter its edit screen, or click Add New at the top to start from scratch.

02

Anatomy of a project page

Every project page is built from three layers, all editable independently:

Title & featured imageThe headline at the top of the page, and the main hero image that loads behind it. Set both at the very top of the edit screen.
Content blocksThe main body of the page — narrative paragraphs, image-text splits, full-width photos, project-data sidebars, pull-quotes, galleries. Stack as many as you like, in any order.
Sidebar metadataThe right-hand info panel: Client, Contractor, Location, Services Provided, Completion Date. Configured via the Project Data block (see section 09).
03

The 7 block types

Each block produces a distinct visual pattern. Mix and match to tell each project's story.

text-paragraph
Narrative
A single block of running prose. Used for storytelling.
full-image
Full image
A wide hero photo with optional caption.
image-pair
Image pair
Two side-by-side images with captions.
image-text-split
Image & text
Photo on one side, narrative on the other.
project-data
Project data
Label/value rows: client, location, scope.
pull-quote
Pull quote
Highlighted testimonial or statement.
gallery
Gallery
Grid of multiple images, 2-4 columns.
04

Adding a block

Below the main content area you'll find the ECEC Content Blocks panel.

Click Add BlockThe button is at the bottom of the block list. A dropdown appears with the seven block types.
Pick a typeEach option has its name and a hint about what it produces.
Fill in the fieldsThe card expands inline with the relevant inputs (image picker, text fields, repeater rows for project-data, etc.).
Save the postClick Update in the top-right. Your block is now live.
✓
Tip:

You can chain as many blocks as you need. The Royal Atlantis Resort and Desert Rock Resort projects use 7 blocks each — that's the recommended depth for a "feature" project. Smaller entries can run with 2-3 blocks.

07

Duplicating & removing blocks

To duplicate a block

Open the block, click the ⋯ menu in the top-right and choose Duplicate. A copy appears directly below.

To remove a block

Click the ⌧ button on the right of the block row. You'll be asked to confirm. The block is removed when you click Update on the post.

i
Undo a deletion.

If you remove a block by mistake, just press Ctrl + Z in the JSON textarea below the block list (advanced). Otherwise, don't click Update — reload the page and the block returns.

08

Images & galleries

Single images

Inside any block with an image field, click the Choose image picker. The WordPress media modal opens. Pick from your existing uploads or drag a new file in. Click Use this image.

Upload area (drag-and-drop ready)

Galleries

The gallery block lets you select multiple images at once. Drag thumbnails inside the block to reorder. Choose 2, 3 or 4 columns from the dropdown.

✓
Recommended specs:

Hero / featured image: 1600 × 900 — landscape 16:9
Full-image block: 1600 × 600 — wide banner
Image-pair / split: 1000 × 700 — portrait or landscape
Gallery thumbs: 800 × 600 — consistent ratio looks best

09

Project data sidebar

The project-data block produces the right-hand info panel on the live project page. It's a list of label/value rows.

Inside the block, you'll see a repeater. Each row has two fields:

  • Label: the bold field name (e.g. Client, Location).
  • Value: the content shown to the right.

Click Add row to insert another. Drag rows to reorder. Empty rows are ignored.

i
Common labels we use:

Client · Contractor · Architect · Location · Sectors · Services Provided · Completion Date · LEED status · Project value

10

Mark as Featured

The home page and the Projects page both show a "Featured" strip pulled from this flag. Toggle it from inside the project edit screen, in the right sidebar under Project options.

Click to toggle

When checked, the project card gets a small ★ FEATURED badge wherever it renders, and the project is included in the home + Projects featured strips. Order is controlled by Menu order: lower numbers appear first.

11

Clone a project as a draft

Have a similar new project to add? Don't rebuild — clone an existing one.

Open the Portfolio admin listHover any project row.
Click Clone as draftAppears alongside Edit / Trash / View. Or use the same option inside the project edit screen, under the Publish box.
You're redirected to the cloneIt's titled "Original (Copy)" and starts as a draft. Title, content, blocks, sidebar data, locations and sectors are all copied across.
Edit and publishChange the title, swap the hero image, edit any text. When ready, change status to Publish.
12

Creating a new project from scratch

  1. Go to WP Admin → Portfolio → Add New.
  2. Type the project title at the top.
  3. In the right sidebar, set the Featured Image — this becomes the hero.
  4. Assign a Sector (Healthcare, Hospitality, etc.) under Portfolio Categories.
  5. Assign a Location (KSA, UAE, Jordan, Other) under Portfolio Locations.
  6. Below the main editor, build out the content blocks (see section 04).
  7. Click Publish.
13

Sectors & locations

Both are taxonomy dropdowns in the right sidebar of the edit screen.

Sectors

10 options: Commercial & Retail, Data Centers, Educational, Governmental Sector, Healthcare, Hospitality, Industrial & Distribution, Master Planning, Residential, Sustainability.

Locations

4 options: Saudi Arabia, UAE, Jordan, Other. Drives the "By Location" dropdown on the Projects search bar.

14

FAQ & gotchas

The block editor looks empty. Nothing happens when I click.

Make sure you're on the portfolio-item edit screen, not a regular page. Try a hard reload: Ctrl + F5. If still empty, clear browser cache for the admin URL.

I deleted a block by mistake.

If you haven't clicked Update yet: just reload the page (F5) and the block returns. After Update, restore via Revisions in the right sidebar — pick a snapshot from before the deletion.

Image caption doesn't show on the live site.

Captions only render if the field is non-empty. Some block types (full-image, image-pair, image-text-split) have a caption field; others (gallery) don't.

Do gallery images open larger when clicked?

Yes — every gallery on every project page is automatically click-to-enlarge (lightbox). Visitors get a fullscreen viewer with prev/next arrows and a counter, can swipe on mobile, and use Esc / arrow keys on desktop. There's no setting to turn on; just upload your gallery images and they're clickable.

What's the default block layout for new projects?

Updated 2026-05-02: text → image-text-split (image LEFT) → image-text-split (image RIGHT) → full-image (1200×400) → project-data → pull-quote → gallery. Previously included an image-pair block which has been replaced by the second image-text-split. You can still add image-pair manually if you need two side-by-side photos.

I cloned a project and both show on the Projects page.

Make sure the clone's status is Draft, not Publish. Drafts don't appear in public listings. Once you've finished editing the clone and want it live, change status to Publish — and you may want to Trash the original if it was just a stepping stone.

I don't see the Location dropdown in the sidebar.

Open Screen Options at the top-right of the edit screen and check the Portfolio Locations box. WordPress hides taxonomy panels by default for new editors.

How do I preview before publishing?

Click Preview in the top-right of the edit screen (above Publish). A new tab opens with the page as it would appear on the live site. Refresh the preview tab after each save.

The Featured strip on the home page is too crowded.

The home page shows the first 3 featured projects (sorted by Menu Order ascending). To rotate which projects appear: open each one, change Menu Order — lowest numbers come first.

How does the Projects page handle a large portfolio?

As of 2026-05-17, the Recent Projects grid on /projects/ shows the first 12 cards (4 rows of 3) and a Load More button below. Clicking the button fetches the next batch via AJAX without a full page reload. New projects you publish automatically slot into the right place by date order — no config needed. To change the per-batch count, open the page in Elementor, find the Recent Projects shortcode, and edit posts_per_page="12".

The titles under featured project cards look cramped.

Fixed 2026-05-17 by adding a small positive letter-spacing override (the parent Qode theme was applying a negative tracking that ran characters together). If you ever notice it again after a theme update, the rule lives in style.css under .ecec-featured-title.
