Adding a new shortcode

  1. Define function ecec_xxx_shortcode( $atts ) in functions.php.
  2. add_shortcode( 'ecec_xxx', 'ecec_xxx_shortcode' );
  3. Conditionally enqueue any per-shortcode JS via wp_enqueue_scripts hook with is_page('xxx') check.
  4. Use filemtime() as the version arg so cache busts on every change (see emaurri_child_theme_enqueue_scripts).

04Custom post types & taxonomies

portfolio-item (theme/plugin CPT)

Provided by Emaurri Core plugin. We extend it with custom meta:

  • _ecec_blocks — JSON array of content blocks (the v2 single-project layout).
  • _ecec_blocks_migrated_at / _ecec_blocks_bulkseeded_at — provenance markers for rollback.
  • _ecec_blocks_backup_premultiblock — pre-rebuild snapshot of _ecec_blocks.
  • _ecec_featured — '1' if shown in home Featured Projects grid.
  • qodef_portfolio_info_items — sidebar facts (Client, Contractor, Services, etc) — theme native.
  • qodef_portfolio_media — gallery images — theme native.

ecec_team_member (custom CPT, in functions.php)

Used by [ecec_team_grid]. Fields:

  • post_title — Name
  • Featured Image — Headshot (square, 220×220 recommended)
  • _ecec_team_role — Role/title (text)
  • _ecec_team_locations — Office locations (text)
  • menu_order — Sort order (lower = first)

portfolio-location (custom taxonomy)

Registered on portfolio-item. Public, REST-enabled. Used by Projects page filter dropdown. Default seeded terms: KSA, UAE, Jordan, Other.

Class auto-applied via WP's post_class(): .portfolio-location-<slug> (no separate JS map needed).

Custom shortcode

06Why ECEC scrollytelling

The Why ECEC page (post 422) renders a single shortcode: [ecec_why]. Sticky left column rotates titles as the right column scrolls.

Architecture

  • Shortcode: ecec_why_shortcode() in functions.php — renders 5 hardcoded pillars in two parallel containers (.ecec-why-info-holder + .ecec-why-main).
  • JS: assets/why-ecec.js — IntersectionObserver picks the slide closest to viewport center, toggles .is-active on the corresponding info item.
  • CSS: .ecec-why-info-holder uses position: sticky; each info item is position: absolute (only one visible via opacity).
  • Sticky workaround: Qode theme's #qodef-page-wrapper has overflow: hidden which breaks position:sticky. Override scoped to body.page-id-422 sets overflow:visible on all relevant ancestors.

Required Elementor data

Post 422 has a single section / column / shortcode widget. Section settings: { layout: 'boxed', padding: {top: 0, ...} }. The full_width layout setting was breaking Elementor's lazy-render on this version — use boxed.

To swap the 5 placeholder images for real ones

  1. Currently the placeholders are hardcoded <div class="ecec-block-placeholder ecec-why-placeholder"> divs in the shortcode.
  2. Quick swap: edit the foreach loop in ecec_why_shortcode() to render <img src="..." /> instead, with URLs uploaded to /wp-content/uploads/why-ecec/.
  3. Better: add an admin option (settings field) that stores 5 attachment IDs, read them in the shortcode. ~30 min refactor.

Reuse this class on any new section heading for visual consistency. letter-spacing + word-spacing are critical — without them uppercase headings read as one merged blob.

Cache busting

functions.php uses filemtime( $style_path ) as the ?ver= arg on the child stylesheet enqueue. Without this, the parent theme defaulted to WP version (e.g. ?ver=6.9.4) which never bumps — browsers cached forever.

Critical

08VPS Elementor schema

This Elementor version uses the LEGACY section + column schema, NOT the modern flexbox container schema. Using elType: 'container' + flex_direction / _flex_size / flex_gap on this VPS causes a fatal Illegal offset type error in Elementor\Core\Files\CSS\Base::get_style_control_value(). I burned ~10 minutes on About Us debugging this.

Correct schema (verified by dumping working pages)

{
  "id": "...",
  "elType": "section",
  "settings": {
    "layout": "boxed",          // or "full_width"
    "structure": "20",          // 2-col 50/50; "30"=3-col, "40"=4-col, omit for 1-col
    "padding": { "unit": "px", "top": "...", ... }
  },
  "elements": [
    {
      "id": "...",
      "elType": "column",
      "settings": {
        "_column_size": 50,           // 100, 50, 33, or 25
        "_inline_size": null,
        "_inline_size_tablet": 100
      },
      "elements": [ /* widgets */ ]
    }
  ]
}

Required post meta for Elementor to render

_elementor_edit_mode MUST be set to 'builder' — without it the page renders with an empty content area even though _elementor_data is populated.

Bug: update_post_meta($pid, '_elementor_edit_mode', 'builder') sometimes silently fails (Elementor save filter intervenes). Reliable fix is direct SQL: 
INSERT INTO wp_postmeta (post_id, meta_key, meta_value)
VALUES (422, '_elementor_edit_mode', 'builder')
ON DUPLICATE KEY UPDATE meta_value='builder';
Always check this first when an Elementor page renders blank.

Things that bit me

09Critical gotchas

1. position: sticky doesn't work

Qode theme's #qodef-page-wrapper has overflow: hidden. This breaks position:sticky on every descendant. Two fixes:

  • For headers: use position: fixed with manual offset (current solution for site header).
  • For per-page sticky elements (Why ECEC): scope overflow: visible !important override on body.page-id-XXX + all wrapper IDs.

2. Theme's .qodef-h2 overrides your h2 color

Theme has h2, .qodef-h2 { color: #000; font-family: "Metric-Semibold"; ... }. Single-class custom selectors lose the cascade. Use !important on color, font-family, font-weight, font-size, text-transform for any custom heading style.

3. Auto-hyphenation breaks word integrity

Per the user's "word-for-word, never summarize" rule, hyphens: auto is FORBIDDEN. Browsers insert visible hyphens mid-word at line breaks (e.g., "engi-\nneering") which breaks the rule. Always use:

hyphens: manual !important;
-webkit-hyphens: manual !important;
word-break: normal;
overflow-wrap: normal;

4. $q->found_posts returns 0 with no_found_rows

If your WP_Query args include 'no_found_rows' => true for performance, $q->found_posts will be 0 (not the actual count). Use count($q->posts) instead. Bit me on the team-grid min padding logic.

5. SCP / Posh-SSH can't handle paths with spaces

Folder names like docs/New Logos/ fail with scp: ambiguous target. Always stage to a no-space path locally before uploading. Bit me with the new logos import — script ran with 0 source files and emptied the ecec_client_logo_ids option (was 39, now 0), breaking the home marquee for ~2 minutes.

6. Defensive: refuse to clear a populated option with empty data

Any import script that REPLACES a non-empty option/setting should refuse to write an empty replacement:

if ( count( $new_ids ) === 0 ) {
    exit "ABORT — no source files found, refusing to clear option";
}

7. RevSlider stores JSON-escaped URLs

When sed-replacing localhost/ecec → 207.180.196.39/ecec in a SQL dump, RevSlider tables also need localhost\/ecec (single backslash) AND localhost\\/ecec (double backslash, in serialized PHP containing JSON). Use _deploy/fix_urls.pl.

grep -c is misleading for verifying — it counts matching LINES, and a single long serialized row can contain hundreds of URL occurrences on one line. Use _deploy/count_urls.pl for real counts.

8. Emaurri Core license bypass

wp-content/plugins/emaurri-core/inc/core-dashboard/class-emaurricore-dashboard.php get_code() must include 207.180.196.39 in its allowed-hosts list. Without this patch, Emaurri Core shortcodes and CPTs don't register on the VPS. Always re-upload after a full DB sync.

9. post_author=0 after DB import

Local exports often have post_author=0 on imported posts → causes Elementor warning + broken get_user_by(). Fix: UPDATE wp_posts SET post_author=1 WHERE post_author=0; after every import.

10. Elementor cache after DB / shortcode change

Elementor caches rendered output in _elementor_element_cache meta + per-post CSS files in wp-content/uploads/elementor/css/post-N.css. Always clear both after editing Elementor data:

DELETE FROM wp_postmeta WHERE post_id=N AND meta_key IN ('_elementor_element_cache','_elementor_css');
rm -f /var/www/html/ecec/wp-content/uploads/elementor/css/post-N.css

11Deployment method

Windows host → VPS via PowerShell + Posh-SSH module (no sshpass). Always invoke with powershell.exe -ExecutionPolicy Bypass.

Pattern

powershell.exe -ExecutionPolicy Bypass -File - <<'PSEOF'
Import-Module Posh-SSH
$pass = ConvertTo-SecureString '<ssh-password>' -AsPlainText -Force
$cred = New-Object System.Management.Automation.PSCredential('root', $pass)
$session = New-SSHSession -ComputerName '207.180.196.39' -Credential $cred -AcceptKey

# 1. Make _tmp dir on VPS
Invoke-SSHCommand -SessionId $session.SessionId -Command 'mkdir -p /var/www/html/ecec/_tmp && chown www-data:www-data /var/www/html/ecec/_tmp' | Out-Null

# 2. SCP files (NO SPACES IN PATHS)
Set-SCPItem -ComputerName '207.180.196.39' -Credential $cred -Path 'D:\xampp\htdocs\ecec\_deploy\my_script.php' -Destination '/var/www/html/ecec/_tmp/' -AcceptKey | Out-Null

# 3. chown + run
$r = Invoke-SSHCommand -SessionId $session.SessionId -Command 'chown www-data:www-data /var/www/html/ecec/_tmp/my_script.php && sudo -u www-data php /var/www/html/ecec/_tmp/my_script.php'
$r.Output

# 4. Cleanup
Invoke-SSHCommand -SessionId $session.SessionId -Command 'rm -rf /var/www/html/ecec/_tmp' | Out-Null
Remove-SSHSession -SessionId $session.SessionId | Out-Null
PSEOF

Rules

  • Always upload to /var/www/html/ecec/_tmp/ (NOT root) so __DIR__ . '/../wp-load.php' resolves.
  • Run scripts as www-data, not root.
  • Clean up _tmp after every deploy.
  • Stage files locally to a no-space path before SCP.
  • For multi-step ops, batch into one PowerShell heredoc — one SSH session round-trip is faster.
