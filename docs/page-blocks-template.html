ECEC Project Page — Block Template Reference

Everything you need to prepare copy and images for a new project page. Seven block types, consistent across all projects, edited inside WordPress admin.

For ECEC Engineering Consultants
Applies to All single-project pages (portfolio-item CPT)
Version v1 · 2026-04-23

Page anatomy

Every project page is composed of a hero section (fixed, driven by the featured image + title) and a body made up of content blocks arranged in order. An info sidebar on the right reads from the existing portfolio info meta (Client, Contractor, Services Provided, Completion Date). Each block is optional — use what the story needs.

HERO
01 · Intro text
02 · Image + Text split
03 · Full-width image (3:1)
04 · Image pair (2 columns)
05 · Project data (facts list)
06 · Pull quote
07 · Gallery (2 or 3 columns)

How it's built

  1. Every block is optional. A short project may use 2-3 blocks; a flagship project 8+.
  2. Block order is controlled from the ECEC Content Blocks editor in the project's admin screen.
  3. The info sidebar is populated from "Portfolio Settings → Info" — unchanged from the existing admin pattern.
  4. The hero is always the featured image + the page title — not a block.
  5. Images are cropped automatically to fit each block's aspect ratio (cover-fit, centred).
Tip: prepare all images as JPG. Use PNG only when transparency is needed (rarely).

01

Text Paragraph

Body copy — intro prose, a narrative paragraph between images, or a closing reflection. Text is justified.

Preview
The engineering brief required systems that respected the dramatic desert terrain while delivering a hospitality experience at the top of the luxury scale. Every decision from MEP routing to thermal comfort was taken in response to the site, not imposed upon it. Guests experience quiet infrastructure; operators experience disciplined redundancy.

Content spec

Text length~80–200 words per block
AlignmentJustified, no hyphenation
Column widthMax 920 px (readable measure)
FormattingPlain paragraphs · **bold**, *italic* OK

Editable fields

body (rich text)

Best used for

Introduction, transitions between visual blocks, closing statement.

02

Image + Text Split

A two-column row: one half image, one half text. Image can be on the left or right. The signature "magazine" block — use it for the most important narrative beats.

Preview
VISION & CONTEXT
A landmark shaped by its landscape
Desert Rock Resort was conceived as a destination that disappears into its canyon setting. Engineering systems were designed to stay invisible while delivering hospitality-grade reliability.

Image spec

Min size1200 × 900 px
Aspect ratio4 : 3 (cover-fit)
FormatJPG

Content spec

Overline2–4 words, uppercase
Heading4–10 words
Body~50–120 words
Image sideLeft (default) or Right

Editable fields

image overline heading body image_side
03

Full-Width Image

A banner-style image that spans the content width. Locked to a 3:1 rectangle — wide, cinematic. Image is cropped automatically to fit.

A dramatic site shot, context view, or completed-building hero. One per page recommended.

04

Two images side-by-side in a 2-column row. Each image has its own optional caption. Great for before/after, exterior/interior, plan/elevation pairs.

06

Pull Quote

A short, italicised statement — a design philosophy, client testimonial, or project-director reflection. Sits inside the body flow as visual punctuation.

Preview

Good engineering is invisible to the guest and indispensable to the operator. On Desert Rock, both happened at the same time.

Project Director, ECEC

Content spec

Quote length10–40 words
AttributionOptional · name + role or organisation

Editable fields

quote attribution

Best used for

One per page max — keep it rare so it feels earned.

07

Gallery

A grid of additional images at the end of the page — detail shots, on-site photography, completed-building exteriors. 2- or 3-column layout.

Preview · 3 columns

Editable fields

images (multi-select) columns (2 or 3)

End-of-page visual summary. Don't dilute — curate 6–9 strong images over 20 average ones.

Quick reference

All block image requirements at a glance. Every image is cover-cropped to fit the block's aspect ratio — centre the subject in your source image.
