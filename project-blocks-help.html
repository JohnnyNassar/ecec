Overview

01What's new

The project editor has been redesigned so you can build rich, magazine-style project pages without touching Elementor. Each project is now assembled from stackable content blocks — seven reusable templates for text, images, quotes, galleries and data. You pick the blocks, drag them into the order you want, and fill in each one.

  • Content Blocks editor — a drag-and-drop card stack below the title
  • Duplicate block — one-click copy of any block, right below the original
  • Clone project — copy an entire project (all blocks, images, info) as a new draft, either from the list or from inside an open project
  • Featured checkbox — promote any project to the home-page highlight grid
  • Location taxonomy — every project is tagged with a region for the Projects-page filter
Heads-up: the legacy project description & media boxes still exist. If the block editor is empty, the old description + media grid renders as a fallback — so no project ever breaks.
Where everything lives

02The project edit screen

When you open or create a project, you'll see three key areas:

  1. Title + permalink — project name at the top.
  2. ECEC Content Blocks — the new block editor (the star of this guide).
  3. Sidebar — on the right: Publish status, Featured checkbox, Categories (Sector), Location, Featured Image, and the legacy Project Info boxes used by the sidebar on the live project page.
Tip: you can collapse/drag any metabox to suit how you work. WordPress remembers your layout per user.
The block editor

03ECEC Content Blocks

This is the metabox labelled ECEC Content Blocks. It's a stack of cards — one card per section of the project page. Each card knows its own type (Text, Image, Gallery, etc.) and its own fields.

The editor mirrors the order of sections on the live project page, top to bottom. Reordering cards here reorders the sections on the site.

Your toolkit

04The 7 block types

Pick the right block for each section of the story you want to tell. You can use each type as many times as you need, in any order.

¶

Text Paragraph

Body copy. Blank lines between paragraphs. Basic HTML (<strong>, <em>, <a>) is allowed.

▭

Full-width Image

One image spanning the full content width, with an optional caption below it.

▭▭

Image Pair

Two images side by side, each with an optional caption. Great for before/after or detail shots.

▭T

Image + Text Split

50/50 image + copy. Overline, heading and body; image on the left or right.

☰

Project Data

Key facts as a label/value table (GFA, Client, Completion, etc.). Drag rows to reorder.

❞

Pull Quote

A large italic quote with an optional attribution. Use sparingly for emphasis.

▦

Gallery

Image grid, 2 or 3 columns. Drag thumbs to reorder, × to remove, "Add images" to append more.

Step-by-step

05Adding a block

  1. Open the Add block: dropdown at the top of the ECEC Content Blocks editor.
  2. Pick the type you want — for example, Image + Text Split.
  3. Click the blue Add button.
  4. The new card appears at the bottom of the list, already expanded, ready for you to fill in.

06Editing a block

Click the Edit link on a card to open it. The card expands to show the fields for its type. Click Close (same position) to collapse again.

Fields save as you type — the JSON behind the scenes updates on every keystroke. You only need to click the WordPress Update button in the sidebar to commit the changes to the database.

Drag to rearrange

07Reordering blocks

Grab the drag handle (the small grid icon on the far left of any card) and drag the card up or down. When you release, the other cards shift around it. The order in the editor is the order on the live project page.

08Duplicating a block

Click Duplicate in any card's header. An exact copy appears immediately below the original, already open for editing. All fields — including images and captions — are copied.

This is perfect when the next section of your project follows the same layout but with different content (e.g. two Image + Text Splits in a row, or several Full-width Images in sequence).

09Removing a block

Click the red × at the end of a card's header. WordPress will ask you to confirm before the block disappears. The change only becomes permanent when you click the Update button in the sidebar.

Undo: if you removed a block and haven't saved yet, just refresh the page — the database still has the old version. If you've already saved, you can restore from the project's Revisions history (WordPress automatically keeps snapshots).
Working with images

10Image picker & gallery

Single images

In blocks with a single image (Full-width Image, Image Pair, Image + Text Split):

  • Click Select image to open the WordPress Media Library.
  • Pick an existing image or upload a new one.
  • After selection, the thumbnail appears in the card with Change image and Remove buttons.

Galleries

In a Gallery block:

  • Click Add images to open the Media Library in multi-select mode — pick as many as you need, they get appended to the end.
  • Drag thumbnails to reorder them inside the gallery.
  • Click the × in the corner of any thumb to remove that image from the gallery (the image stays in your Media Library; it's just not used in this block anymore).
  • Toggle between 2 columns and 3 columns with the columns dropdown.

Facts & figures

11Project data rows

The Project Data block is a table of label + value pairs — perfect for the "at a glance" strip with GFA, client, completion date, certifications, etc.

  • Each row has a Label (e.g. "GFA") and a Value (e.g. "56,000 sqm").
  • Click Add row under the list to create another pair.
  • Drag the row handle on the left to reorder rows.
  • Click the × on a row to remove just that row.

The block also has an Overline (small caps above the heading, usually "PROJECT DATA") and a Heading (e.g. "At a glance") — both are optional.

Home-page promotion

12Mark as Featured

In the right-hand sidebar on the project edit screen, find the Featured project checkbox. Tick it, then click Update. The project now shows up in the Featured section at the top of the Projects page, and a small ★ FEATURED badge appears on its card wherever it's rendered.

Order of featured projects follows the project's menu order number (WordPress's built-in sort field, in the Page Attributes box). Lower numbers show first. Set menu order 1 on your flagship, 2 on the next, and so on.
Keep it curated: 4 to 8 featured projects works best visually. More than that and the Featured section loses its emphasis.
Copy a whole project

13Clone a project

Cloning creates a new project as a draft with everything copied over: blocks, featured image, categories, location, sidebar info, project data rows — the lot. Useful when a new project is similar in structure to an existing one (e.g. two hospitality projects with the same data fields).

There are two places to start a clone:

From the Projects admin list

Go to Portfolio in the left admin menu. Hover over any row — a set of small action links appears under the title: Edit, Quick Edit, Trash, View, and the new Clone as draft.

From inside the project edit screen

Unsaved changes don't clone: if you start a clone from the edit screen while you have unsaved changes, those changes won't be copied — the clone reads from the last saved version. Save first if your edits matter.
From scratch

14Create a new project

If you're starting fresh (rather than cloning):

  1. From the admin menu, click Portfolio → Add New.
  2. Type the project title at the top.
  3. Assign the right Category (Sector) and Location in the right-hand sidebar.
  4. Upload a Featured Image — this becomes the hero of the single project page and the thumbnail everywhere else.
  5. Fill in the Project Info metabox (Client, Contractor, Engineering, Completion Date, etc.) — these drive the sidebar on the live page.
  6. Scroll to ECEC Content Blocks. Pick a block type from the dropdown and click Add to create your first section. Repeat until the page tells the right story.
  7. Click Publish.
Recommended first blocks for any project: a short Text Paragraph to introduce the project, then a Full-width Image or Image + Text Split for context, then a Project Data block for key stats, then a Gallery at the end.
Good to know

15Tips & troubleshooting

The block editor looks empty / nothing happens when I click

The block UI depends on JavaScript. Try a hard refresh (Ctrl+Shift+R on Windows, ⌘+Shift+R on Mac). If it still doesn't appear, check that you're logged in with an account that can edit posts.

I deleted a block by mistake

If you haven't clicked Update yet, just refresh the page. Your last saved version still has the block. If you've already saved, go to the project edit page and look at the Revisions link in the Publish box — pick a previous version to restore.

Image caption doesn't show on the live site

Captions only render when they're not empty. Double-check the Caption field in the block. Keep captions short — a photographer credit or location line works best.

The Featured section looks too busy

Unfeature some projects (uncheck the Featured box) or tighten the menu-order numbers to decide which appear first. Aim for 4-8 at a time.

I cloned a project and both show up on the Projects page

They won't — clones are created as drafts. Only published projects appear on the site. The clone stays invisible until you publish it.

I don't see the Location dropdown in the sidebar

Scroll further down the right-hand column, or click Screen Options at the top-right of the WordPress admin and tick the Location checkbox.

The block summary doesn't match what I typed

Summaries update on every keystroke. If they look stale, try clicking into another field and back — that forces a refresh. Also, the summary only reads plain text (no HTML) for readability.

How do I preview before publishing?

Use the Preview Changes button in the Publish box. It opens the draft project in a new tab as it would appear on the live site, without actually publishing.

Site-wide

16Replacing placeholder images

Across the site you'll see gray boxes with a dashed border like  600 × 800  — those mark spots where a real image hasn't been uploaded yet. They're intentional reminders. Below is how to replace them in each location.

1. Project page images (most common)

The placeholders inside image-text-split, full-image, image-pair, and gallery blocks are managed via the project editor — same flow as section 10:

  1. Open the project: Projects → click the project name.
  2. Find the block in the Content Blocks editor (the gray placeholder shows where the image will appear).
  3. Click the empty image slot or the Choose image button → upload from your computer or pick from the media library.
  4. Click Update at the top of the page. The placeholder disappears the moment a real image is set.

2. About Us page (2 placeholders)

The two large gray boxes in the body of the About Us page are placed via Elementor (the visual page editor). To swap them:

  1. Go to Pages → click About Us.
  2. Click the blue Edit with Elementor button at the top.
  3. In the Elementor canvas, click on the gray placeholder box. The left panel will show its settings.
  4. The placeholder is currently an HTML widget. To replace with a real image: delete the HTML widget, then drag an Image widget from the left panel into the same column, click it, and choose your image.
  5. Click the green Update button bottom-left.

If you'd like the About Us images to be swappable from the standard WordPress admin (not Elementor), ask your developer — that's a one-time refactor we can do.

3. Why ECEC page (5 sliding placeholders)

The five gray boxes on the Why ECEC page are part of a custom scrolling layout and currently can't be swapped from the WordPress admin. Send your 5 images to your developer and they'll wire them in once. After that swap, the placeholders disappear and the real images appear in the same scroll-cycle.

Recommended image specs: portrait orientation, ~600 × 800 px or larger, JPEG.

4. People page (placeholder team cards)

The gray cards under the team grid are auto-placeholders — they appear only when there are fewer than 6 published team members. To remove them, just add more real team members:

  1. Team Members in the left admin menu → Add New.
  2. Fill in the name, role, and locations; upload a square headshot as the Featured Image.
  3. Set the order number (lower = appears first), then Publish.

As soon as you've published 6 team members, the placeholder cards disappear automatically. You can publish more than 6 — extras just continue the grid.

5. Services page (4 section images)

The Services page has four image slots in the various 2-column sections. Same flow as About Us:

  1. Pages → Our Services (opens directly in Elementor).
  2. Click the placeholder image in the canvas.
  3. Use the left panel's image picker to upload or select your replacement.
  4. Update bottom-left.

Image specs cheat-sheet

Recommended pixel sizes by location (the system will resize down — bigger is OK, smaller will look blurry):

  • Project blocks — full-image: 1200 × 400 (3:1 wide)
  • Project blocks — image-text-split / image-pair: 600 × 450 (4:3)
  • Project blocks — gallery: 400 × 300 (4:3)
  • About Us / Services 2-col placeholders: 600 × 700 (portrait)
  • Why ECEC slides: 600 × 800 (portrait)
  • Team headshot: 220 × 220 (square)
  • Hero / page-title bar (e.g. About Us chairs photo): 1920 × 470 (very wide)

If something looks wrong after upload

  • Refresh with Ctrl+Shift+R — sometimes the browser caches the old placeholder.
  • Check that the image is large enough (under 800 px wide can look blurry on big screens).
  • If the layout shifts oddly, the image's aspect ratio is very different from the placeholder. The system fits the image to the slot but if it's extreme (e.g. very tall portrait into a wide slot) it may crop unexpectedly.
